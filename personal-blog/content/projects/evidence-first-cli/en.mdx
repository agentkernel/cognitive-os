<SampleNotice />

<h2 id="problem">Problem</h2>

The sample command-line tool needs to show long-running task state, while remote receipts and model text cannot become local acceptance.

<h2 id="constraints">Constraints</h2>

The Shell only consumes projections; exiting cannot imply cancellation; an unknown Effect must stay visible.

<h2 id="approach">Approach</h2>

Separate `watch`, `cancel request`, and `acceptance` as command semantics, then attach a source, version, and next gate to every displayed state.

<h2 id="outcome">Outcome</h2>

The sample information architecture clearly demonstrates the difference between received, candidate complete, and accepted. There is no real implementation or user result here.

<h2 id="reflection">Reflection</h2>

Client copy is part of the governance boundary. A premature “complete” label is more dangerous than missing animation.
