<h2 id="problem">Problem</h2>

A bilingual documentation migration can leave broken language switches, automatically generated but unstable heading anchors, and mobile navigation that only works with a pointer.

<h2 id="constraints">Constraints</h2>

Page bodies remain server-rendered; only mobile navigation holds necessary client state; both languages share explicit anchors.

<h2 id="approach">Approach</h2>

Create an explicit content registry, pairing tests, and visible focus styles, then cover skip links, Escape-to-close, and 44-pixel touch targets in browser tests.

<h2 id="outcome">Outcome</h2>

The sample demonstrates an acceptance method, but includes no real migration scope, team, or performance figures.

<h2 id="reflection">Reflection</h2>

Accessibility is not a scan before release. It belongs in the design of routes, content identity, and interaction state.
